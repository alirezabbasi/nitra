# BUG-00007: OANDA/Capital Connectors Degraded After No-Mock Cutover

## Status

- Resolved in code
- Runtime rebuild/restart required

## Description

After no-mock ingestion cutover, non-Coinbase connectors remained degraded:

- OANDA returned persistent `404` due malformed pricing URL assembly.
- Capital repeatedly re-authenticated and hit `429`, then stalled on `404` epic lookups.
- Coinbase Exchange ticker returned `400` for some products without fallback handling.

## Impact

- Runtime ingest appeared to be "Coinbase only" in practice.
- OANDA and Capital emitted no usable quotes.
- Connector health remained degraded and produced noisy error loops.

## Reproduction

1. Rebuild and start compose ingestion runtime.
2. Check connector logs:
   - OANDA shows `/pricing/stream/v3/accounts/.../pricing` malformed URL path.
   - Capital shows repeated `POST /api/v1/session` `429` and `GET /prices/<epic>` `404`.
3. Query latest ingestion rows and observe no fresh OANDA/Capital flow.

## Root Cause Analysis

- OANDA API base derived from `OANDA_STREAM_URL` by host replacement only, preserving stream path suffix and duplicating `/v3/accounts/.../pricing`.
- Capital logic treated all quote-fetch errors as auth failures, forcing re-login on non-auth statuses (`404`) and amplifying login rate-limit failures (`429`).
- Coinbase logic had no fallback path when Exchange ticker endpoint rejected specific products/runtime contexts.

## Resolution Implemented

- Updated OANDA base URL handling to:
  - honor `OANDA_REST_URL` when provided,
  - sanitize stream URL-derived host by dropping `/v3/...` path before building REST pricing URL.
- Updated Capital quote fetch/error handling to:
  - treat `404` as "epic not found" and continue candidate resolution,
  - re-authenticate only on auth-expiry statuses (`401/403`),
  - avoid fail-fast on per-epic non-auth errors.
- Added Coinbase fallback to `COINBASE_PUBLIC_REST_URL /v2/prices/{product}/spot` when Exchange ticker is unavailable (`400/403/404`).
- Updated compose and env contracts for connector-specific URL/map variables.
