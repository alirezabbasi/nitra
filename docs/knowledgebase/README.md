# Knowledgebase

This folder stores practical incident knowledge and post-fix operating guidance.

## Structure

- `charting/` chart runtime/UI incidents and fixes.
- Future domains can be added as sibling folders (for example `ingestion/`, `normalization/`, `ops/`).

## Record Format

Each knowledgebase record should include:

1. `Problem`
2. `Symptoms`
3. `RCA`
4. `Resolution`
5. `Validation`
6. `Guardrails`

## Naming

Use: `KB-xxxxx-short-title.md`.
