# BUG-00009 - Liquidity Layer diverges from ontology-driven M5 closed-candle engine outputs

## Status

Open

## Date

2026-05-01

## Impact

- Chart `Liquidity Layer` can display minor/major pullback annotations that do not match canonical ontology behavior.
- Overlay updates intra-candle from live ticks instead of strictly on closed `M5` candles.
- Structure shown on chart is not sourced from deterministic `structure-engine` outputs, causing trust mismatch between visual layer and engine state.

## Reproduction

1. Open chart page with explicit context, e.g. `/charting?venue=oanda&symbol=GBPUSD&timeframe=1m`.
2. Enable `Liquidity Layer`.
3. Compare chart overlay behavior against expected ontology-based M5 closed-candle rules.
4. Observe that overlay updates with live tick polling and uses client-side modeled pairs within 48h rolling window.

## Evidence

- `services/charting/static/index.html`
  - `buildLiquidityStructureModel` computes minor/major pairs entirely client-side from displayed bars.
  - `resolveLiquidityBias` uses last-close position in rolling range (`pos > 0.55`) rather than ontology directional liquidity state contract.
  - `getLiquidityWindowBars` uses fixed `LIQ_LAYER_WINDOW_MS = 48h` rolling window.
  - `syncLiquidityLayer` summary explicitly reports `Window:48h`.
  - `pollLiveTicks` calls `updateChartDataIncremental(...)` and triggers `syncLiquidityLayer()` on live tick updates, not only closed candles.
- Runtime/compose evidence:
  - `docker-compose.yml`: `structure-engine` input topic is `bar.1m` (not dedicated `5m` closure stream).
  - DB query: `structure_state` currently only has `timeframe = '1m'`.

## Root Cause

- Liquidity layer was implemented as a front-end heuristic overlay (`DEV-00054`) for visualization parity, not as a projection of ontology-compliant deterministic engine outputs.
- No contract binds chart overlay to structure-engine minor/major events scoped to current-day + previous-day closed `M5` candles.

## Corrective Direction

1. Introduce backend endpoint for ontology-derived liquidity pairs sourced from deterministic engine outputs (or engine-read DB projection), scoped to `M5` and `today + yesterday`.
2. Update chart liquidity overlay to consume that endpoint instead of local `buildLiquidityStructureModel`.
3. Gate overlay refresh to closed `M5` candle boundaries (or server-published closed-candle events), not per-tick.
4. Add deterministic tests for:
   - inside/outside bar precedence,
   - pullback start/termination,
   - minor/major archive semantics,
   - M5 closed-candle update cadence.

## Verification Needed After Fix

- For a selected symbol, overlay minor/major pairs match ontology fixture replay over `today + yesterday` window.
- No overlay mutation occurs between M5 candle closes.
- Charted pairs match persisted structure outputs for same venue/symbol/timeframe window.
