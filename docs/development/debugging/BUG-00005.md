# BUG-00005: Coinbase "Live" Feed Is Still Mock-Generated

## Status

- Resolved in code
- Runtime restart required

## Description

`market-ingestion` currently emits synthetic/randomized quotes for all venues, including Coinbase.
As a result, Coinbase prices in `raw_tick`/charting can diverge from actual exchange market prices.

## Impact

- "Live ingestion" validation is not actually validating live venue market data.
- Price comparisons against external Coinbase quotes fail by design.
- Downstream bars/backfill/replay are based on synthetic prices, not venue truth.

## Reproduction

1. Run compose ingestion stack with `market-ingestion-coinbase`.
2. Query latest Coinbase ticks in `raw_tick`.
3. Observe `source = nitra.market_ingestion.mock` and synthetic bid/ask/mid generation.

## Root Cause Analysis

- `services/market-ingestion/src/main.rs` uses random walk generation for quotes and always stamps:
  - `"source": "nitra.market_ingestion.mock"`
- Connector envs (`COINBASE_WS_URL`, `OANDA_STREAM_URL`, `CAPITAL_*`) are present in compose but not used by runtime ingestion logic.

## Resolution Implemented

- Replaced synthetic/random quote generation in `services/market-ingestion/src/main.rs` with venue-sourced fetch paths:
  - Coinbase ticker endpoint
  - OANDA pricing API
  - Capital authenticated price endpoint
- Enforced fail-closed behavior for `CONNECTOR_MODE=mock`.
- Removed runtime mock pricing module (`services/ingestion/mock_pricing.py`).
- Added test guardrails (`tests/dev-00005/run.sh`, `tests/dev-00009/run.sh`) to block mock-source strings and random-generation paths.

## Verification Evidence

- Live DB sample (2026-04-24 UTC) shows Coinbase ticks with mock source:
  - `source = nitra.market_ingestion.mock` for latest `BTCUSD/ETHUSD/SOLUSD/ADAUSD/XRPUSD`.
- Runtime code path confirms synthetic generation loop and fixed mock source tag in:
  - `services/market-ingestion/src/main.rs`.
