# BUG-00008: OANDA/Capital Candles Missing from `ohlcv_bar` Despite `raw_tick` Ingestion

## Status

- Resolved in code
- Runtime rebuild/restart required

## Description

`raw_tick` ingestion was healthy for all venues (`oanda`, `capital`, `coinbase`), but `ohlcv_bar` contained only Coinbase candles.

## Impact

- OANDA and Capital charts showed no fresh OHLCV bars.
- Multi-venue chart continuity and analytics were degraded.
- Gap/backfill orchestration could be triggered unnecessarily for non-Coinbase venues.

## Reproduction

1. Start compose runtime and allow ingestion to run for several minutes.
2. Verify `raw_tick` rows exist for `oanda`, `capital`, and `coinbase`.
3. Query `ohlcv_bar` grouped by venue and observe only Coinbase rows.
4. Inspect normalized events and observe stale `event_ts_exchange` values for OANDA/Capital (around `20:58`/`20:59` UTC) while receive-time advances.

## Root Cause Analysis

- `bar-aggregation` bucketed bars strictly by `event_ts_exchange`.
- OANDA/Capital connectors emitted stale/frozen `event_ts_exchange` values for quote events.
- Since the bucket never advanced for those venues, in-memory bars were never rolled/flushed into `ohlcv_bar`.

## Resolution Implemented

- Updated `bar-aggregation` to select event time with sanity checks:
  - use `event_ts_exchange` when fresh,
  - fallback to `event_ts_received` when exchange timestamp is stale or far-future.
- Added unit tests in `services/bar-aggregation/src/main.rs` for fresh vs stale timestamp behavior.
- Added SDLC guard in `tests/dev-0012/run.sh` to assert fallback logic presence.

## Verification Evidence

- `cargo test --manifest-path services/bar-aggregation/Cargo.toml`
- `tests/dev-0012/run.sh`
- Live runtime verification after rebuild:
  - `ohlcv_bar` now contains `capital` and `oanda` rows alongside `coinbase`,
  - per-symbol venue rows observed for:
    - `capital`: `EURUSD`, `GBPUSD`, `NAS100`, `US30`, `XAUUSD`
    - `oanda`: `EURUSD`, `GBPUSD`, `USDJPY`, `NAS100`, `US30`, `XAUUSD`
