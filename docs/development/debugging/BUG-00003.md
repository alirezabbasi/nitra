# BUG-00003: Mock Ingestion Published Incorrect Uniform Prices Across Instruments

## Status

- Resolved in code
- Runtime restart required

## Description

In mock ingestion mode, all configured instruments shared a single FX-like price stream starting around `1.085`.
This caused crypto instruments (for example `BTCUSD`, `ETHUSD`) to be published with unrealistic prices and near-identical movement patterns.

## Impact

- `raw_tick` data for multi-asset runs was materially incorrect.
- `ohlcv_bar` generated from these ticks inherited unrealistic price levels.
- Charting/validation queries showed misleading values and false cross-instrument correlation.

## Reproduction

1. Run ingestion stack in current mock mode with multiple symbols (for example `EURUSD,BTCUSD,ETHUSD`).
2. Query latest rows in `raw_tick` grouped by `broker_symbol`.
3. Observe all symbols tracking around the same level and shape.

## Root Cause Analysis

`services/market-ingestion/app.py` used one shared variable:

- `price = 1.0850`

Then generated one `bid/ask/mid` per loop and reused those values for every symbol.
No per-symbol state, baseline, spread, or volatility profile existed.

## Resolution Implemented

Code fixes:

- `services/market-ingestion/app.py`
  - added per-symbol price state and symbol-specific initial baselines
  - added asset-aware spread and step sizing (FX vs crypto)
  - added symbol-aware quote precision (`EURUSD`, `USDJPY`, crypto)
- `services/ingestion/domain.py`
  - added `infer_asset_class_from_symbol()` helper
- `services/market-normalization/app.py`
  - switched `asset_class` from constant `fx` to derived classification from canonical symbol

## Verification Evidence

- Added test pack: `tests/dev-00009/run.sh`
  - validates distinct base behavior for FX vs crypto symbols
  - validates symbol precision/step/spread logic expectations
- Existing ingestion verification (`tests/dev-00005/run.sh`) still passes.

## Notes

- This fix addresses mock-mode data realism for development/testing.
- Real venue connectors should still replace mock generation for production-like validation.
