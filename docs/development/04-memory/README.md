# Project Memory System

This section is NITRA's persistent project memory.
It is designed to survive long gaps between sessions without losing direction.

## Files

- `MEMORY_OPERATING_SYSTEM.md`: update rules and lifecycle.
- `WHERE_ARE_WE.md`: concise status snapshot for rapid progress checks.
- `CURRENT_STATE.md`: authoritative latest project state snapshot.
- `DECISION_LOG.md`: durable architecture/product/implementation decisions.
- `RISKS_AND_ASSUMPTIONS.md`: active risk register and assumptions.
- `SESSION_LEDGER.md`: chronological session handoff log.
- `templates/`: standard templates for consistent memory updates.

## Update discipline

Any substantial implementation/planning session must update:

1. `CURRENT_STATE.md`
2. `WHERE_ARE_WE.md`
3. `SESSION_LEDGER.md`
4. `DECISION_LOG.md` and/or `RISKS_AND_ASSUMPTIONS.md` when changed
