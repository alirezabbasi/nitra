# Governance

This section defines how NITRA work is planned, executed, reviewed, and resumed.

## Files

- `SDLC_OPERATING_MODEL.md`: execution workflow, role split, quality gates, and update cadence.

## Non-negotiables

- HLD alignment check before implementation and before closure.
- Step-based traceability: implementation -> tests -> docs.
- No memory drift: every substantial session updates `04-memory/` artifacts.
